Public Endpoints:



GroupMembers:


https://api.irenebot.com/members (GET)-> View minimal information about ALL members
https://api.irenebot.com/members_with_photos (GET)-> View minimal information about ALL members WITH photos.
https://api.irenebot.com/members/:id (GET)-> View minimal information about a specific idol.
https://api.irenebot.com/groups (GET)-> View information about all groups.
https://api.irenebot.com/groups/:id (GET)-> View information about a specific group from their group id.
https://api.irenebot.com/:idolID/list (GET)-> Get all the file ids an idol has.
https://api.irenebot.com/downloaded (GET)-> Get ~1000 random file names that are downloaded.
https://api.irenebot.com/botinfo (GET)-> Get total commands used, server count, member count, and idol photo count.
https://api.irenebot.com/idolcommandsused (GET)-> Get amount of idol photo commands used.
https://api.irenebot.com/invite (GET)-> Link to invite bot to server.
https://api.irenebot.com/patreon (GET)-> Link to patreon page.


https://api.irenebot.com/random/ (GET)-> Get information on a random image of an Idol.
Status 404 -> Idol has no Photos, this should not happen unless the database is empty.
Status 500 -> Server Issue
Status 200 -> Will return JSON below. Status 415 (does not match the actual http status) -> Is a video file, purposed for IreneBot, will also return a JSON.
Example (200, 415):

{
'final_image_link': 'https://www.images.irenebot.com/idol/3132image.jpeg',
'location': '/var/www/public_html/3132image.jpeg',
'file_name': '3132image.jpeg'
}

Private Key Endpoints

Translation:

Pass in Private Keys as an Authorization Bearer Token in the header.


https://irenebot.com/translate (POST)-> Send in 'text' to translate, 'src_lang', 'target_lang' as a body.
Note that translate does not use the api sub-domain.

GroupMembers:

Pass in Private Keys as an Authorization Bearer Token in the header.


https://api.irenebot.com/photos/:idolID (POST)-> Get redirected to an image of the Idol. Pass in 'allow_group_photos' as 1 to get any photos, or 0 to not get group photos.
Status 404 -> Idol has no Photos
Status 500 -> Server Issue
Status 200 -> You have been redirected to the image (essentially 308). If using from code and not postman, you can typically get the url of the open connection and that will be the image url.
Status 415 (does not match the actual http status) -> Is a video file, purposed for IreneBot, it will return a JSON.
Example:

{
'final_image_link': 'https://www.images.irenebot.com/idol/3132image.jpeg',
'location': '/var/www/public_html/3132image.jpeg',
'file_name': '3132image.jpeg'
}

Pass in Private Keys as an Authorization Bearer Token in the header.


https://api.irenebot.com/photos/guessing_game/ (POST)-> Pass in 'allow_group_photos' as 1 to get any photos, or 0 to not get group photos.
Status 500 -> Server Issue
Status 200 -> Will receive a json containing Idol Information along with the image url.
Example:

{
'member_id': 1,
'full_name': "Bae Joo Hyun",
'stage_name': "Irene",
'image_url': 'https://www.images.irenebot.com/idol/3132image.jpeg',
'aliases': ['joohyun', ...]
}

https://api.irenebot.com/file/:imageID (POST)-> Get redirected to an image of a file id.
Status 404 -> Image ID does not exist.
Status 500 -> Server Issue
Status 200 -> You have been redirected to the image (essentially 308). If using from code and not postman, you can typically get the url of the open connection and that will be the image url.
Status 415 (does not match the actual http status) -> Same as /photos/:idolID endpoint.
